About

Solara

System Architect · Warsaw, Poland

15 years old. Self-taught. Based in Warsaw. Building for around 16 months. I don't write code directly — I design systems and architectures, hand implementation to AI, and own the review and deploy steps myself.

What I care about is the part that doesn't fit in a single file: how requests flow across the network, where state lives, what fails when, and what the operator sees when it does. My codebases live mostly in Rust — a custom container orchestrator, a custom reverse proxy, a custom scheduler, a toy VCS — plus a Django SaaS that exists to pay for the rest of the stack.

The web side is deliberately old-fashioned: vanilla HTML, CSS, and JavaScript. No React, no Tailwind, no Node, no build step. The maps and panels you're reading right now are about 700 lines of plain JS.

Stack

Linux kernel networking — namespaces, veth pairs, bridges, iptables. Docker via bollard. AWS EC2. CI/CD as GitHub Actions → rsync → deploy. Observability with Prometheus, Grafana, Loki, Promtail, Tempo, OpenTelemetry, Alertmanager.

System design

Comfortable with the standard distributed-systems vocabulary — CAP, consensus, BFT, consistent hashing, replication, partitioning, distributed transactions, napkin math.
