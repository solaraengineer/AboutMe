004 / Data

Databases

Knowing which database to use is more important than knowing how to use it.

A Typical Conversation

YOU

"We have lots of data. We need more database."

ME

"What kind of data? How are you querying it?"

YOU

"Just... lots of it. Assets. Tiles. Things that get rendered."

ME

Cassandra.

YOU

"Why?"

ME

"Super fast I/O for reads and writes. Designed for massive throughput. Perfect for large data assets that don't change often and need to be served fast."

YOU

"But what about our user auth? And the relational stuff?"

ME

"Yeah, Cassandra isn't for that. Keep Postgres for auth and relational data. Use the right tool for each job. Cassandra for assets, Postgres for auth, Redis for caching. That's the architecture."

PostgreSQL Relational

My primary database. ACID compliance, complex queries, proper foreign keys. When data integrity matters, Postgres is the answer.

Use for: User data, transactions, anything with relationships

Redis Cache

In-memory speed. Session storage, rate limiting, Celery broker, real-time pub/sub. The glue that makes everything faster.

Use for: Caching, sessions, rate limits, message queues

Cassandra NoSQL

Massive throughput for read-heavy workloads. When you need to serve large assets fast and don't need relational queries.

Use for: Large assets, logs, data that doesn't change often
